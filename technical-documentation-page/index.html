Introduction

Cascading Style Sheets (CSS) is a language for writing style sheets, commonly used to describe how HTML documents should be rendered on screen, paper, speech, or other media. By separating content from design, CSS allows developers to apply styles—such as colors, fonts, and layouts—consistently across multiple pages. This separation enhances both the efficiency and maintainability of web design. More specifically:

  • A style sheet is a set of rules that specify the presentation of a document. Style sheets are written by an Author (developer), and interpreted by a User-Agent (browser), to present the document to the User (person using the website). For instance, I wrote a styles.css file to apply to this site and gave it to Chrome. It then applies my specified theme and presents the stylized page to you. For precise definitions, please visit the World Wide Web Consortium (W3C) specification.
  • Styles are applied to the source document in HTML or XML (or XML dialects).
  • A cascade algorithm determines what style to apply to an element. It takes input from multiple stylesheets, including ones from the User-Agent (i.e. default browser settings), the Author, and the User. For each element on the HTML document, it gathers the rules written for that element and computes the precedence of each rule. It finally applies the highest-precedence rule to the element.

Beyond custom website themes, CSS offers powerful features like media queries for responsive design and flexbox or grid layouts for complex arrangements. It is among the core languages of the open web and is standardized across Web browsers. Thus, mastering CSS is essential for creating visually appealing and user-friendly websites.

Maintenance

CSS is maintained by the W3C. Previously, its modules were developed synchronously, which allowed versioning (see: CSS1, CSS2.1). As progress on different modules began to vary, W3C shifted to releasing separate recommendations for each module. Now, the modules have individual versions instead of the language as a whole, such as “CSS Color Module 5.” W3C publishes a “snapshot” of the latest stable state of the specification at regular intervals.

Basic rule syntax

The building blocks of CSS statements include declarations, selectors, and at-rules.

Declarations

Setting CSS properties to specific values is the core function of the CSS language. A property and value pair is called a declaration, and any CSS engine calculates which declarations to apply to every element on the webpage.

Both properties and values are case-insensitive by default in CSS. The pair is separated by a colon, as shown below.

There are hundreds of different properties and an endless number of values. Some combinations may be deemed invalid and thus ignored.

For example, the following is an invalid declaration, so the CSS engine will not apply this rule:

background-color: zero; /* invalid value */

Rulesets

Group declarations by using curly braces to create declaration blocks. To specify the elements that the block of rules applies to, we add a list of selectors—which are conditions selecting some elements of the page—to the beginning. This forms a CSS ruleset.

As an element of the page may be matched by several selectors, and therefore by several rules potentially containing a given property several times, with different values, the CSS standard defines which one has precedence over the other and must be applied: the cascade algorithm.

Note: If one selector is invalid, the entire ruleset is ignored.

Statements

There are two kinds of statements:

  • Rulesets that associate a collection of CSS declarations to a condition described by a selector.
  • At-rulesAt-rules that start with an 'at' sign, '@', followed by an identifier, and then continuing up to the end of the statement. Each type is defined by the identifier, and may vary in internal syntax. They are used to convey meta-data information (like @layer or @import), conditional information (like @media or @document), or descriptive information (like @font-face).

Rulesets may be nested in a conditional at-rule:

@media (max-width: 700px) { /* nested rulesets */ }
Selectors

Basic HTML element types can be selected, along with user-defined ids, classes, and attributes. IDs are referenced with a hashtag ‘#’ in front of the identifier, while class selectors are constructed using a period ‘.’ in the beginning. Attribute selectors follow the format “[attr=value]”. The universal selector, ‘*’, selects all the elements on the page.

Examples of my favorite selectors so far include:

  • The span[class~="sr-only"] attribute selector specifies only span elements whose class includes "sr-only."
  • The :first-of-type pseudo-selector is used to target the first element that matches the selector.
  • The :nth-of-type(number here) pseudo-selector is used to target specific elements based on their order among siblings of the same type.
  • :hover = hover effect.
  • With the descendant selector .class descendant, elements matched by the second selector are selected if they have an ancestor (parent, parent's parent, parent's parent's parent, etc.) element matching the first selector.

For an exhaustive list, please visit the corresponding MDN site.

Properties

There are hundreds of CSS properties. Here are my favorites:

  • The position property determines the position of the element on the page. The different positions include:
    • static: default flow of elements onto page
    • relative: follows default flow, but activates top, left, etc properties
    • absolute: removed from normal document flow but positioned according to nearest containing block
    • fixed: same as absolute but fixed according to viewport
    • sticky: toggles between relative and fixed, depending on the scroll position, like the “freeze view” in Google Sheets.
  • The transform property allows you to modify the shape, position, and size of an element without changing the layout or affecting the surrounding elements. It has functions such as translate(), rotate(), scale(), skew(), and matrix(). For example, the following rule rotates the box by -45 degrees:
    • .box { transform: rotate(-45deg) }
  • Setting the overflow property to hidden effectively hides any sub-element that is pushed outside the set width of the object.

For an exhaustive list, please visit the corresponding MDN site.
